## Instructions

1. **Read and understand** the task completely before starting
2. **Ask questions** if ANYTHING is unclear - do this BEFORE writing code
3. **Follow TDD** - write the failing test first, watch it fail, then implement
4. **Write minimal code** to make the test pass - no over-engineering
5. **Self-review** your work before reporting completion:
   - Does it match the spec exactly?
   - Are there any edge cases missed?
   - Is the code clean and well-named?
   - Did you follow the project's conventions?
6. **Commit** with a descriptive message

## Output

When complete, report:
- What you implemented
- What tests you wrote and their results
- Self-review findings (anything you noticed)
- The commit SHA

If you have questions, ask them FIRST before starting implementation.
